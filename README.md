# Naicibox

Terminal-centric dev environment toolkit — per-project isolation with a single setup command.

---

## Goals

- Clean project isolation (history, aliases, env vars)
- Fast and keyboard-driven (tmux + Bash + Starship)
- Claude Code integration per project
- Single-script provisioning via `setup.sh`
- Zero naicibox files committed to project repos

---

## Tools Used

| Tool              | Purpose                                |
|-------------------|----------------------------------------|
| **tmux**          | Multi-window terminal sessions         |
| **Bash**          | Shell with per-project config          |
| **VS Code**       | Editor launched from project context   |
| **Starship**      | Fast, minimal, informative prompt      |
| **Claude Code**   | AI coding assistant per project        |
| **direnv**        | Auto-load environment vars per folder  |

---

## Prerequisites

- **Node.js / npm** — required for Claude Code installation

---

## Architecture

Project repos stay clean. Naicibox config lives separately in `PROJECTS_HOME/.naicibox/`:

```
~/projects/                     (PROJECTS_HOME — your private naicibox_projects repo)
├── .naicibox/
│   ├── myapp/
│   │   ├── .bashrc             # project-specific aliases
│   │   └── starship.toml       # project-specific prompt
│   └── otherapp/
│       └── ...
├── myapp/                      # clean git repo
│   └── .envrc                  # auto-generated, gitignored
└── otherapp/
```

The `.envrc` in each project is generated by `newproject` and should be gitignored. It sources config from `.naicibox/` and sets up per-project history and Starship prompt.

Track `PROJECTS_HOME` as a private repo to back up all your per-project config.

## Usage

### 1. Deploy Environment

```bash
./setup.sh
```

Installs packages (`git`, `curl`, `tmux`, `direnv`, `starship`, `code`, `claude`), deploys dotfiles, and configures your shell with `NAICIBOX_HOME`, `PROJECTS_HOME`, and `bin/` on PATH. Open a new terminal after running.

### 2. Create a New Project

```bash
newproject myproject
```

Creates `$PROJECTS_HOME/myproject/` and `$PROJECTS_HOME/.naicibox/myproject/` with config files. Generates `.envrc` in the project and runs `direnv allow`.

To adopt an existing repo, run `newproject` from inside it:

```bash
cd ~/path/to/existing-repo
newproject existing-repo
```

If the current directory name matches the project name, `newproject` sets up naicibox config without creating a new directory.

### 3. Start a Project

```bash
project-start.sh myproject
```

Launches VS Code and a tmux session with Claude Code on top and a shell on the bottom.

### 4. Per-Project Starship Prompt

Each project gets its own `starship.toml` in `.naicibox/<project>/`. Cloud/infra modules (AWS, GCloud, Azure, Kubernetes, Docker, Terraform) are disabled by default. Language detectors (Python, Node, Rust, Go, etc.) work automatically.

To enable a module for a specific project, edit `.naicibox/myproject/starship.toml`:

```toml
[aws]
disabled = false
```

### 5. Per-Project Aliases

Edit `.naicibox/myproject/.bashrc` to add project-specific aliases:

```bash
alias build='make -j$(nproc)'
alias serve='python3 -m http.server'
```

## What's Included

- `setup.sh` — single entry point for environment provisioning
- `files/` — base `.bashrc`, `.bash_aliases`, and `.tmux.conf`
- `template/config/` — per-project config copied to `.naicibox/<project>/`
- `template/envrc` — `.envrc` template generated into each project
- `bin/` — helper scripts: `project-start.sh`, `newproject`
